News

2005-12-30: I finally came around to install a first demonstration. You can view it here. For testing there are two staff-users 'teststaff' and 'teststaff2' (password equals username). It's quite a complex example of how you can use germ. There's also a diagram of the ER model.

  • lpms source code: 'cf.py' contains configuration variables used by germ, 'entity' contains the entity-relationship specification, 'htdocs' contains the html template.
  • /cgi-bin/lpms.py: This is the start-up script. It executes germ and points it to lpms.

Installation

germ is still under development and not ready for end users. If you want to try it nevertheless you need the following:
  • A webserver which allows execution of cgi scripts
  • A mysql database server
  • You also need pso, a set of python scripts which germ uses for web session management. I simply downloaded their latest release.
  • You can get germ via CVS or by downloading the prerelease

CVS access

cvs -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/germ login
cvs -z3 -d:pserver:anonymous@cvs.sourceforge.net:/cvsroot/germ co -P germ
